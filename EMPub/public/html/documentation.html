Introduction

The introduction section of Software Requirements Specification (SRS) outlines the purpose, scope, document conventions, intended audience, and an overview of the SRS. The main purpose of this paper is to provide a complete description of EMPub system by formulating the main problems it solves; providing feasible solutions to the problem.

1.1 Purpose

The Epub application is the software that is described in this paper. It is an extension of the standard mailer. Using this application, a mail user can easily share a message from his inbox, providing access to it to multiple friends in just one click.

Throughout the documentation, we will present different use cases, and requirements in respect to the users and the system, and also outline the concepts that may be developed later. The paper specifies the projectâ€™s main audience, user interfaces, user interaction with the interface, hardware and software requirements.

1.2 Scope

The EMPub application focus on making messages from a mail client inbox public, so it can be seen by an unlimited or limited number of users. One of the most common use cases of our application is the situation in which a user what to forward a received email to a large group of colleagues. The current solution to this problem is adding in receive field of the letter email address of all colleagues. Therefore, we want to provide a platform in which the previous operation will take just a few clicks.

1.3 Intendent audience

This paper is intended to be read by developers and testers as detailed information will be provided in respect to the implementation of the application functionality, however, certain chapters of our documentation could be easily read by simple users. The second chapter Overall description describes application functionality and presents UML diagrams, and the first paragraph of the third chapter describes Users Interfaces are the documentation part intended for user reading.

SYSTEM FEATURES

4.1 Receiving and publishing emails

Through this feature the product receive emails from mailing services (Gmail , Yahoo, etc.) and publish them on the user email page. The publication is made by sending a forward to an address controlled by the web application. Posting parameters can be changed by a tag system included in the body of the email for example: [password = password] [duration = 2h10m] [public].

4.2 Publication list manager

Through this feature a user can view and manipulate the list of his publications and his posting parameters : expiration date, sender, subject, availability (public/private), number of views. The list of publications can be ordered by availability, number of views, expiration date and time. An email can be searched by sender or subject.

4.3 Email settings manager

Through this feature a user can change the default publishing parameters (public, password protected, expiration date and time, etc.) or can delete an email through the web interface.

4.4 Email statistics

Through this feature a user can view the list of related statistics (number of views, country of origin of visitors, schedule of visits by time intervals) for a period viewed (per day, per week, per month) chosen by the user. Statistics will be available in HTML, PDF and XML formats.

4.5 Login via email

hrough this feature a user account is created upon receipt of the first email from an email address or via the web interface. Authentication in the web application will be done by temporary passwords sent to the email address of the account.
